### 2.2 `display_window.py` ⚠️ 存在缺陷

| 检查项 | 结果 | 说明 |
|--------|------|------|
| Pygame 独立线程 | ✅ 通过 | 使用 `threading.Thread` + `daemon=True` |
| Always on Top | ✅ 通过 | `user32.SetWindowPos(hwnd, -1, ...)` 正确 |
| 线程锁 | ✅ 通过 | `threading.Lock` 保护共享参数 |
| 窗口停止 | ✅ 通过 | `stop()` 使用 `join(timeout=3.0)` |

**缺陷 #D1（中等）— 窗口尺寸检测逻辑错误**

```python
# 当前代码（第 ~ 行）：
w, h = self._screen.get_size()          # ← 先读取当前窗口尺寸
with self._lock:
    if w != self._target_w or h != self._target_h:   # ← 但比较的是已被覆盖的 w, h！
        ...
```

**问题**：在 `with self._lock` 外部已经执行了 `w, h = self._screen.get_size()`，在 `with self._lock` 内部读取的是 `_target_w, _target_h`，逻辑正确。但之后立即用 `_target_w, _target_h` 覆盖了 `w, h`，导致下一次循环迭代时 `w, h` 已经是目标值，`get_size()` 的结果永远不会被比较。

**实际影响**：窗口尺寸**可以正常设置**（`set_mode` 被调用），但检测逻辑的意图被破坏。若在测量中途外部代码调用了 `set_size()`，可能会被正确响应，但若 Pygame 内部改变了尺寸（理论上不太可能），检测会失效。

**建议修复**：
```python
# 方案一：在锁内读取
with self._lock:
    target_w, target_h = self._target_w, self._target_h
w, h = self._screen.get_size()
if w != target_w or h != target_h:
    with self._lock:
        self._screen = pygame.display.set_mode(
            (self._target_w, self._target_h), pygame.NOFRAME
        )
    self._apply_always_on_top()

# 方案二：保留当前读取顺序，删除覆盖行
```

### 2.4 `measurement_controller.py` ⚠️ 白色模式逻辑不符合 PRD

| 检查项 | 结果 | 说明 |
|--------|------|------|
| 状态机转换 | ✅ 通过 | 8 个状态完整实现，转换路径正确 |
| 中止机制 | ✅ 通过 | `_abort.set()` + 循环内双重检查 |
| 白色 RGB 同值 | ✅ 通过 | 单色模式 `r = g = b = gray` |
| **白色模式灰阶逻辑** | ❌ **不符合 PRD** | 见缺陷 #D2 |

**缺陷 #D2（严重）— 白色模式灰阶逻辑与 PRD 不符**

PRD §2.2 明确要求：
> **白色模式**：循环顺序：R 通道灰阶递增（G=B=0）→ G 通道灰阶递增（R=B=0）→ B 通道灰阶递增（R=G=0）
> 示例（start=0, end=2）：R0 → R1 → R2 → G0 → G1 → G2 → B0 → B1 → B2（共 9 次测量）

当前实现（`_do_work` 第 ~230 行）：
```python
# Set display color: white = RGB same value; single-color = RGB same value
r = g = b = gray
if self.display_window:
    self.display_window.set_color(r, g, b)
```

白色模式与单色模式的处理**完全相同**，均输出 `RGB = (gray, gray, gray)`。

**PRD 要求 vs 当前实现对比**：

| 场景 | PRD 要求 | 当前实现 |
|------|----------|----------|
| 白色, start=0, end=2 | 9 次测量（R0→R2, G0→G2, B0→B2） | 3 次测量（白0→白1→白2） |
| 通道字段 | 'R'/'G'/'B'/'单色' | '单色'（固定） |

**影响**：白色模式的测量结果不符合产品定义。若产线使用白色模式测试，将无法分别获得 R/G/B 三通道的独立色坐标数据。

**建议修复**：
```python
# 白色模式：三通道独立递增
if color_name == MeasurementController.COLOR_WHITE:
    channels = ['R', 'G', 'B']
    for ch in channels:
        for gray in range(start_gray, end_gray + 1):
            if ch == 'R': r, g, b = gray, 0, 0
            elif ch == 'G': r, g, b = 0, gray, 0
            else: r, g, b = 0, 0, gray
            # 测量...
            record['通道'] = ch
else:
    # 单色模式（红/绿/蓝）
    for gray in range(start_gray, end_gray + 1):
        r = g = b = gray
        # 测量...
        record['通道'] = '单色'
```

**缺陷 #D3（轻微）— CSV 记录中通道字段固定为"单色"**

CSV 字段 `通道` 在白色模式下应区分 'R'/'G'/'B'，但当前固定为 '单色'：
```python
'通道': '单色',  # white mode: single RGB value
```
建议随 D2 一并修复。
