│           │                                              │

│  │            serial_protocol.py                     │   │
│  │            (串口通讯层 / pyserial)                 │   │

│  ┌────────▼─────────────────────────────────────────┐   │

│  └──────────────────────────────────────────────────┘   │
└─────────────────────────────────────────────────────────┘
```

**文字描述：**
- **主窗口**（PySide6）和**打屏窗口**（Pygame）是两个独立窗口，通过 Qt 信号/曹机制传递状态和数据
- **measurement_controller** 是核心控制层，包含状态机和工作线程，负责测量流程编排
- **serial_protocol** 是底层串口封装，负责指令收发和 CR 分隔符处理
- 所有串口操作在工作线程中执行，结果通过 Qt 信号派发回主线程更新 UI

```

## 3. 串口协议解析方案

### 3.1 分隔符处理（CR `\r`）

CA-410 使用 CR (`\r`, 0x0D) 作为指令分隔符和回复分隔符。

```python
# serial_protocol.py — 核心读取逻辑
import serial

class CA410Protocol:
    def __init__(self, port: str, baudrate=9600, timeout=3.0):
        self.serial = serial.Serial(
            port=port,
            baudrate=baudrate,
            bytesize=serial.EIGHTBITS,
            stopbits=serial.STOPBITS_ONE,
            parity=serial.PARITY_NONE,
            timeout=timeout
        )
        self.serial.flushInput()

    def send_command(self, cmd: str) -> str:
        """发送指令并读取 CR 分隔的回复"""
        # 1. 发送：指令 + CR
        cmd_bytes = cmd.encode('ascii') + b'\r'
        self.serial.write(cmd_bytes)
        self.serial.flush()

        # 2. 读取：直到遇到 CR
        buffer = bytearray()
        while True:
            byte = self.serial.read(1)
            if byte == b'\r' or byte == b'':  # 读到 CR 或超时
                break
            buffer.extend(byte)

        return buffer.decode('ascii').strip()
```

**关键设计**：
- `timeout=3.0`（3000ms）自动处理超时，无需手动设置 readline 长度
- `strip()` 去除首尾空白，保证 `OK00` 等关键字段可直接比较
- 读超时时 `read(1)` 返回空字节 `b''`，触发循环退出

### 3.2 回复匹配逻辑

```python
    def expect_ok(self, cmd: str) -> bool:
        """发送指令并验证 OK00 回复"""
        reply = self.send_command(cmd)
        return reply.startswith('OK00')

    def parse_measurement(self, reply: str) -> dict | None:
        """解析 MES,1 回复，格式: OK00,P1,0,{x},{y},{lv},..."""
        if not reply.startswith('OK00'):
            return None
        parts = reply.split(',')
        if len(parts) < 6:
            return None
        return {
            'x': float(parts[3]),
            'y': float(parts[4]),
            'lv': float(parts[5])
        }
```

**匹配策略**：字符串 `startswith('OK00')` 优于正则表达式，简洁高效且满足协议需求。

### 3.3 串口扫描

```python
    @staticmethod
    def scan_ports() -> list[str]:
        """扫描并返回符合关键词的串口列表"""
        import serial.tools.list_ports
        keywords = ['Measuring Instruments', 'USB 串行设备']
        matches = []
        for port in serial.tools.list_ports.comports():
            name = port.descriptive_port_name
            if any(kw in name for kw in keywords):
                matches.append(port.device)
        return matches
```

## 4. 状态机详细设计

### 4.1 状态定义

```python
class State(IntEnum):
    IDLE        = 0   # 就绪，等待用户操作
    SCANNING    = 1   # 正在扫描串口
    CONNECTING  = 2   # 正在连接（发送 COM,1）
    CALIBRATING = 3   # 正在校准（发送 ZRC）
    MEASURING   = 4   # 正在循环测量
    ABORTING    = 5   # 正在中止（等待循环退出）
    EXPORTING   = 6   # 正在导出 CSV
    ERROR       = 99  # 异常状态
```

### 4.2 状态转换表

| 当前状态 | 触发事件 | 目标状态 | 执行动作 |
|----------|----------|----------|----------|
| IDLE | `start_measurement` | SCANNING | 禁用控件；调用 `scan_ports()` |
| SCANNING | 设备找到 | CONNECTING | 打开串口；发送 `COM,1` |
| SCANNING | 设备未找到 | ERROR | 弹窗"色度仪未连接" |
| CONNECTING | 回复 OK00 | CALIBRATING | 发送 `ZRC` |
| CONNECTING | 其他回复/超时 | ERROR | 弹窗"色度仪通讯失败" |
| CALIBRATING | 回复 OK00 | MEASURING | 进入测量循环 |
| CALIBRATING | 其他回复/超时 | ERROR | 弹窗"色度仪校准失败" |
| MEASURING | 测量完成 | EXPORTING | 发送 `COM,0`；进入导出 |
| MEASURING | 测量指令失败 | ERROR | 弹窗"第N灰阶测量失败" |
| MEASURING | `abort_measurement` | ABORTING | 设置 `self._abort_flag = True` |
| ABORTING | 循环退出 | EXPORTING | 发送 `COM,0`；进入导出 |
| EXPORTING | CSV 导出完成 | IDLE | 恢复控件；记录日志 |
| ERROR | 任意 | IDLE | 恢复控件；记录日志 |

### 4.3 信号/事件机制

```python
class MeasurementController(QObject, StateMachine):
    # === Qt Signals（工作线程 → GUI 主线程）===
    state_changed = Signal(str)          # 状态名变化
    log_message   = Signal(str)         # 日志消息
    data_ready    = Signal(dict)         # 单条测量数据
    progress_updated = Signal(int, int) # (当前灰阶, 总灰阶)
    finished      = Signal(str)          # 完成（含文件路径或错误）
    abort_confirmed = Signal()           # 中止确认

    # === 公有方法（GUI 主线程调用）===
    def start(self, params: dict): ...
    def abort(self): ...
    def get_current_state(self) -> State: ...
```

**Qt Signal/Slot vs threading.Event**：
- 选 PySide6 内置的 `Signal/Slot`，因为数据（测量记录、日志）需要跨线程传递
- `threading.Event` 仅用于工作线程内部的**中止标志**（`_abort_event.is_set()`）

## 5. 线程模型

### 5.1 三线程职责

```
┌──────────────────────────────────────────────────────┐
│  主线程（PySide6）                                    │
│  - 运行 Qt 事件循环（QApplication.exec）              │
│  - 处理所有 UI 事件（按钮点击、参数输入）               │
│  - 通过 Qt Signal/Slot 接收工作线程数据                │
│  - 绝对不直接访问 pyserial                            │
└──────────────────────────────────────────────────────┘
                              ▲ Qt Signal
┌──────────────────────────────▼──────────────────────┐
│  工作线程（MeasurementThread，QThread）                │
│  - 串口扫描（scan_ports，主线程也可做，此处放工作线程）  │
│  - 发送指令、接收回复（所有串口 I/O）                   │
│  - 测量循环：打屏→休眠 100ms→发送 MES,1→解析数据       │
│  - CSV 导出（文件 I/O）                                │
│  - 通过 Signal 将数据派发给主线程                      │
└──────────────────────────────────────────────────────┘
                              ▲ 直接调用（无锁需求）
┌──────────────────────────────▼──────────────────────┐
│  打屏线程（DisplayThread，Pygame 独立线程）            │
│  - Pygame.init() 和 pygame.event.get() 独立事件循环   │
│  - 仅做全屏色块渲染，无数据竞争                        │
│  - 由主线程调用 set_color() 更新目标颜色               │
│  - （可选：合并到主线程用 QOpenGLWidget 做打屏）        │
└──────────────────────────────────────────────────────┘
```

### 5.2 线程间通信

```python
# 主窗口 → 工作线程：通过公有方法（线程安全，因为 GIL + Qt 内部锁）
controller.start(params)    # 主线程调用
controller.abort()          # 主线程调用

# 工作线程 → 主线程：通过 Qt Signal（Qt 内部队列，自动线程安全）
controller.data_ready.emit(record)   # 在工作线程中 emit
# → 主线程 Signal Slot 接收：self.controller.data_ready.connect(self.on_data_ready)

# 线程安全原则：所有 pyserial 操作在工作线程；所有 QWidget 操作在主线程
```

### 5.3 中止机制

```python
class MeasurementThread(QThread):
    def __init__(self):
        self._abort_event = threading.Event()  # 工作线程内部中止标志

    def abort(self):
        self._abort_event.set()   # 主线程调用（threading.Event 本身线程安全）

    def run(self):
        for gray_level in range(start, end + 1):
            # 每次循环开始检查中止标志
            if self._abort_event.is_set():
                self.log('测量已中止')
                break

            self.set_display_color(gray_level)
            time.sleep(0.1)  # 100ms 测量间隔
            self.send_measure_command()

            # 同样在 sleep 中检查（确保最多延迟 100ms 响应中止）
            if self._abort_event.is_set():
                break
```

**中止响应延迟**：最多 100ms（一个测量间隔），满足产线场景需求。

## 8. 尚需评审讨论的问题

### Q1：打屏窗口技术选型（Pygame vs QOpenGLWidget）

**问题**：PRD 要求打屏窗口为独立顶层窗口（Always on Top），当前设计采用 Pygame 独立线程渲染。

**两个方案**：

| 方案 | 优点 | 缺点 |
|------|------|------|
| **A：Pygame 独立线程**（当前推荐） | 渲染独立，不受 Qt 事件循环阻塞；代码隔离清晰 | 需跨线程传递颜色参数（用 `queue.Queue`） |
| **B：QOpenGLWidget / QWindow** | 与主窗口同一渲染体系，减少跨线程开销 | 若 Qt 事件循环卡顿，打屏刷新也会受影响 |

**建议**：采用方案 A（Pygame），保证打屏渲染的实时性，不受测量流程中大量信号/日志更新的影响。

### Q2：白色模式灰阶逻辑最终确认

PRD 正文中「颜色灰阶逻辑」描述了分通道 R→G→B 逻辑，但 PRD 末尾「尚需确认」表格中明确：**白色模式 = RGB 同值（直接打白色灰阶图）**，且 PRD 中 "2.2 颜色灰阶逻辑" 小节内容与"已确认"结论矛盾。

**结论（依据确认表）**：白色模式下，RGB 三通道同值递增，伪代码：

```python
for gray in range(start_gray, end_gray + 1):
    set_color(R=gray, G=gray, B=gray)   # 白色 = RGB 同值
    measure()
```

**总测量次数** = `end_gray - start_gray + 1`

请再次确认此理解是否正确。
