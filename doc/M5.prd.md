## 1. 产品概述

### 1.1 产品名称
**ca-analyzer**（色度分析仪）

### 1.2 产品定位
ca-analyzer 是一款面向产线测试场景的色度测量工具，对接柯尼卡美能达 CA-410 色度仪，通过可配置的打灰阶方式循环测量色坐标（x/y）和亮度值（Lv），自动导出 CSV 数据报表，大幅提升显示模组色彩标定效率。

### 1.3 M5 模块定位
M5 是 ca-analyzer 的核心测量模块，负责：
- 串口通讯与色度仪控制（探测、校准、测量、关闭）
- 打屏窗口的实时渲染与预览
- 灰阶循环测量与数据采集
- CSV 测量报告导出

> **备注**：本项目为全新代码库，与 led-test-tool 无直接关联，后续如需整合可作为独立模块接入。

### 1.4 关键技术指标

## 2. 功能需求详述

### 2.1 输入参数列表

| # | 参数名 | 中文名 | 类型 | 默认值 | 取值范围 | 校验规则 |
|---|--------|--------|------|--------|----------|----------|
| 1 | `offset_x` | 打屏起始 X 坐标 | int | 0 | 0 ~ 屏幕最大宽度 | ≥ 0 |
| 2 | `offset_y` | 打屏起始 Y 坐标 | int | 0 | 0 ~ 屏幕最大高度 | ≥ 0 |
| 3 | `width` | 打屏宽度 | int | 512 | 1 ~ 屏幕最大宽度 | ≥ 1 |
| 4 | `height` | 打屏高度 | int | 512 | 1 ~ 屏幕最大高度 | ≥ 1 |
| 5 | `start_gray` | 打屏开始灰阶 | int | 0 | 0 ~ 255 | 0 ≤ start_gray ≤ end_gray |
| 6 | `end_gray` | 打屏结束灰阶 | int | 255 | 0 ~ 255 | start_gray ≤ end_gray ≤ 255 |
| 7 | `color` | 打屏颜色 | enum | 白色(#FFFFFF) | 红色(#FF0000)、绿色(#00FF00)、蓝色(#0000FF)、白色(#FFFFFF) | 单选项 |

### 2.2 颜色灰阶逻辑（关键）

#### 白色模式（RGB 三通道独立递增）
- 循环顺序：R 通道灰阶递增（G=B=0）→ G 通道灰阶递增（R=B=0）→ B 通道灰阶递增（R=G=0）
- 示例（start=0, end=2）：R0 → R1 → R2 → G0 → G1 → G2 → B0 → B1 → B2（共 9 次测量）
- 灰阶步进：每次 +1

#### 单色模式（红/绿/蓝）
- 选定颜色从 start_gray 到 end_gray 递增，R/G/B 同值
- 示例（红色, start=0, end=2）：R0 → R1 → R2（共 3 次测量）
- 灰阶步进：每次 +1

### 2.3 按钮控件

| 控件 | 功能 |
|------|------|
| 【测量】按钮 | 启动测量流程 |
| 【中止】按钮 | 停止当前测量循环 |

    ▼

    │
    ▼
[参数实时预览] ──→ [打屏窗口实时更新颜色和规格]
    │
    ▼

    │
    ├─[Step 1] 禁用所有参数输入 & 禁用测量按钮 & 启用中止按钮
    │
    ├─[Step 2] 串口扫描
    │   ├─ 发现有效设备 → 继续

    │
    ├─[Step 3] 发送探测指令 COM,1
    │   ├─ 回复 OK00 → 继续
    │   └─ 其他回复 → 报错「色度仪通讯失败」→ 跳转 Step 7
    │
    ├─[Step 4] 发送校准指令 ZRC
    │   ├─ 回复 OK00 → 继续
    │   └─ 其他回复 → 报错「色度仪校准失败」→ 跳转 Step 7
    │
    ├─[Step 5] 循环打屏 + 测量
    │   │
    │   ├─ [循环体]
    │   │   ├─ 打屏窗口绘制当前颜色+灰阶 → 休眠 100ms
    │   │   ├─ 发送测量指令 MES,1
    │   │   │   ├─ 收到 OK00,... → 解析 x, y, Lv → 记录数据
    │   │   │   └─ 其他回复 → 报错「第N灰阶测量失败」→ 跳转 Step 7
    │   │   └─ 灰阶/通道递增，继续下一循环
    │   │
    │   └─ [循环结束条件]
    │       ├─ 正常结束（灰阶递增完毕）
    │       └─ 中止触发（用户点击中止）
    │
    ├─[Step 6] 发送结束指令 COM,0
    │   ├─ 回复 OK00 → 继续
    │   └─ 其他回复 → 报错「色度仪关闭失败」（仍继续导出 CSV）
    │
    ├─[Step 7] 导出 CSV
    │   └─ 生成文件：色度测量-{start}-{end}-{timestamp}.csv

## 5. 串口通讯协议

### 5.1 串口扫描规则

扫描所有可用串口，匹配以下条件的视为 CA-410 色度仪：
```
port_object.descriptivePortName 包含 "Measuring Instruments"
  或
port_object.descriptivePortName 包含 "USB 串行设备"
```

> **注意**：实际 CA-410 可能使用虚拟串口驱动，名称以设备管理器显示为准。

### 5.2 指令格式

> **已确认**：分隔符为 CR (`\r`)，指令以回车符结尾。

| 步骤 | 指令名称 | 发送命令（示例） | 成功回复（示例） | 失败处理 |
|------|----------|-----------------|-----------------|----------|
| 探测 | 打开通讯 | `COM,1\r` | `OK00\r` | 其他回复 → 通讯失败 |
| 校准 | 色度校准 | `ZRC\r` | `OK00\r` | 其他回复 → 校准失败 |
| 测量 | 获取测量值 | `MES,1\r` | `OK00,P1,0,{x},{y},{lv},...` | 其他回复 → 测量失败 |
| 关闭 | 关闭通讯 | `COM,0\r` | `OK00\r` | 记录警告，继续执行 |

### 5.3 回复数据解析

成功测量回复格式：
```
OK00,P1,0,x,y,lv,...
```
按逗号 `,` 分割后：

| 索引 | 字段 | 示例值 | 说明 |
|------|------|--------|------|
| 0 | 状态标志 | `OK00` | 成功标识 |
| 1 | 数据类型 | `P1` | （协议定义） |
| 2 | 测量编号 | `0` | |
| 3 | x | `0.3274345` | CIE x 色坐标 |
| 4 | y | `0.4191236` | CIE y 色坐标 |
| 5 | lv | `4.8075729` | 亮度值（cd/m²） |
| 6+ | 其他 | `0.39,...` | 可忽略 |

### 5.4 串口参数建议

| 参数 | 推荐值 |
|------|--------|
| 波特率 | 9600（CA-410 默认） |
| 数据位 | 8 |
| 停止位 | 1 |
| 校验位 | None |
| 分隔符 | CR (`\r`，0x0D） |
| 超时时间 | 3000ms（测量指令） |

## 6. 数据处理逻辑

### 6.1 白色模式（直接打白色灰阶图）

```
总测量次数 = end_gray - start_gray + 1

伪代码:
for gray in range(start_gray, end_gray + 1):
    set_color(R=gray, G=gray, B=gray)   # 白色 = RGB 同值
    measure()
```

> ⚠️ **白色模式**：RGB 三通道同值，直接打白色灰阶图，**不是**分通道 R→G→B 依次测量。

### 6.2 单色模式（红/绿/蓝）

```
总测量次数 = end_gray - start_gray + 1

伪代码:
for gray in range(start_gray, end_gray + 1):
    set_color(R=gray, G=gray, B=gray)   # 灰阶 = RGB 同值
    measure()
```

### 6.3 数据记录结构

```python
measurement_record = {
    "index": int,       # 测量序号（从 1 开始）
    "gray": int,        # 当前灰阶值
    "channel": str,     # 当前通道（R/G/B/单色）
    "color": str,       # 颜色名称（红/绿/蓝/白）
    "x": float,         # CIE x 色坐标
    "y": float,         # CIE y 色坐标
    "lv": float,        # 亮度值
    "timestamp": str    # 当前时间戳（YYYY-MM-DD HH:mm:ss）
}
```

### 7.3 异常处理策略

| 异常场景 | 处理方式 | 状态转换 |
|----------|----------|----------|
| 串口扫描无设备 | 弹窗报错 + 日志记录 | SCANNING → ERROR → IDLE |
| 探测指令超时/失败 | 弹窗报错 + 日志记录 | CONNECTING → ERROR → IDLE |
| 校准指令超时/失败 | 弹窗报错 + 日志记录 | CALIBRATING → ERROR → IDLE |
| 测量指令超时/失败 | 弹窗报错 + 日志记录 | MEASURING → ERROR → IDLE |
| 关闭指令失败 | 记录警告日志，不中止 | 任意 → EXPORTING |
| 用户点击中止 | 退出循环，进入导出 | MEASURING → ABORTING → EXPORTING |

## 8. CSV 导出格式

### 8.1 文件命名规则

```
色度测量-{start_gray}-{end_gray}-{timestamp}.csv
```

其中 `timestamp` 格式：`YYYYMMDD_HHmmss`
示例：`色度测量-0-255-20260417_114530.csv`

### 8.2 保存路径

> **已确认**：弹出系统另存为对话框（`QFileDialog.getSaveFileName`），用户自行选择保存路径和文件名。

### 8.3 CSV 内容格式

```csv
序号,灰阶值,通道,颜色,x,y,亮度Lv,测量时间
1,0,R,白,0.3274345,0.4191236,4.8075729,2026-04-17 11:45:30
2,1,R,白,0.3280000,0.4200000,5.1000000,2026-04-17 11:45:31
...
```

| 列名 | 数据类型 | 说明 |
|------|----------|------|
| 序号 | int | 从 1 开始的测量序号 |
| 灰阶值 | int | 当前灰阶（0~255） |
| 通道 | str | 当前通道（R/G/B/单色） |
| 颜色 | str | 颜色名称（红/绿/蓝/白） |
| x | float | CIE x 色坐标 |
| y | float | CIE y 色坐标 |
| 亮度Lv | float | 亮度值（cd/m²） |
| 测量时间 | datetime | 精确到秒的测量时刻 |
